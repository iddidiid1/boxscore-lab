# Backend

Express + TypeScript API for BoxScore Lab, the personal fantasy basketball MVP.

See the root [README](../../README.md) for setup and running instructions, and
[docs/](../../docs/) for operating and design documentation.

## Structure

```txt
src/
├─ app/        # Express app assembly, routes, server bootstrap, startup checks
├─ modules/    # business modules (see below)
├─ shared/     # db (Prisma client), errors, utils, constants
├─ config/     # env loading and config object
└─ main.ts     # entry point
```

## Modules

Each module owns its routes, controller, service, and validation:

- `divisions`
- `teams`
- `players`
- `events`
- `matches`
- `health` (`GET /api/health` → `{ "status": "ok" }`)

All routes are mounted under `/api/*` in `src/app/routes.ts`.

## Key behavior

- Statistics and team points are computed at request time from raw records — never stored. See
  [docs/TEAM_POINTS_RULES.md](../../docs/TEAM_POINTS_RULES.md).
- `Team.slug` / `Player.slug` are generated by the backend and are immutable.
- API errors use a consistent shape: `{ statusCode, error, message, details }`, with all validation
  errors collected before responding.

## Commands

```bash
pnpm dev:backend                                   # dev server (tsx watch)
pnpm --filter @fantasy-league-stats/backend build
pnpm --filter @fantasy-league-stats/backend typecheck
```

## MVP boundaries

No authentication, authorization, registration, automatic scheduling, or tournament-bracket logic.
