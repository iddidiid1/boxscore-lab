does not decide how a specific Team, Event, Match, Player, or other business

## Inventory rules

- Inventory only current MVP flows and already-established product behavior.
- A feature may produce multiple UI items.
- Merge visual duplicates only after confirming that their states and semantics
  are genuinely compatible.
- Do not force a product pattern into a reusable component prematurely.
- Record where an item is used before redesigning it.
- Track non-happy states, not only the default specimen.

## Fields

| Field | Meaning |
|---|---|
| ID | Stable identifier used by batches and decisions |
| Item | Human-readable component or pattern name |
| Level | Foundation, Component, Pattern, or Scenario |
| Current usage | Pages/features where it currently appears |
| Required states | States the experiment must cover |
| Responsive | Required mobile or breakpoint behavior |
| Dependencies | Tokens or lower-level components it consumes |
| Status | Workflow status defined in `README.md` |
| Decision | Link or identifier from `DECISION_LOG.md` |
| Notes | Scope, exception, or unresolved question |

## E2 batch contracts

### B01 — Foundation

- **Real usage:** global canvas/sidebar/content surfaces; headings and dense
  sports data; controls, tables, cards, overlays, and responsive layouts.
- **States/cases:** approved E0 palette, complete type roles, density steps,
  radius/border hierarchy, focus-visible, hover/pressed/selected foundations,
  reduced motion, elevation, desktop/mobile overflow, and non-color signals.

  colors remain approved, while derived roles may be tuned through specimens.
- **Exceptions:** data-driven Team colors and later Tier/radar/award/winner
  visualization roles.
- **Out of scope:** finished Button, Form, Table, or product-card styling.

### B02 — Surfaces and content hierarchy

- **Subphase B02-A:** compare Open Section, Solid Tonal, Editorial Outline,
  Dark Glass, Dark Neumorphism, and Gradient Surface using identical content,
  geometry, and base/active states. Determine whether materials are global
  defaults, bounded-container options, or named feature exceptions.
- **Subphase B02-B:** after material review, resume containment, page/section
  hierarchy, Inset Content Panel, Compact Metadata Fact, and eyebrow decisions.
- **Real usage:** overview/create/edit page headers; section headings; standard,
  inset, and compact metadata containers.
- **States/cases:** base and active/hover material response,
  title/description/action combinations, nested surfaces, compact facts, long
  content, fallback without backdrop filtering, action wrapping, and narrow
  layouts.
- **Candidate token impact:** panel/inset/compact surface roles, content widths,
  padding, heading roles, dividers, contextual radius, translucency, blur,
  material highlights, gradient stops, and restrained shadow roles.
- **Exceptions:** Detail pages do not receive the Functional Page Header.

### B03 — Actions and navigation
